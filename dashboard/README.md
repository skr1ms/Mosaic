# Dashboard - Система управления купонами и партнерами

## 🚀 Быстрый старт

### 1. Запуск приложения

```bash
# Установка зависимостей
npm install

# Запуск в режиме разработки
npm start
```

Приложение будет доступно по адресу: `http://localhost:3001`

### 2. Запуск Backend (обязательно!)

**Важно:** Dashboard требует запущенный backend сервер!

```bash
# Перейдите в папку backend
cd ../backend

# Запустите backend сервер
go run cmd/main.go
# или
go run .
```

Backend должен быть доступен по адресу: `http://localhost:8080`

### 3. Вход в систему

При первом запуске вы увидите страницу входа в систему. Для тестирования доступны следующие учетные данные:

#### 👨‍💼 Администратор
- **Логин:** `admin`
- **Пароль:** `admin123`
- **Доступ:** Полный доступ ко всем функциям системы

#### 🤝 Партнер
- **Логин:** `quicktest`
- **Пароль:** `Partner123!`
- **Доступ:** Только главная панель и аналитика

### 4. Диагностика проблем

#### Проблема: "Не удается подключиться к серверу"

1. **Проверьте, что backend запущен:**
   ```bash
   # В папке backend
   go run cmd/main.go
   ```

2. **Проверьте, что backend доступен:**
   ```bash
   curl http://localhost:8080/api/login/admin
   ```

3. **Проверьте консоль браузера:**
   - Откройте DevTools (F12)
   - Перейдите на вкладку Console
   - Посмотрите на ошибки подключения

#### Проблема: "Ошибка входа. Проверьте логин и пароль"

1. **Проверьте тестовые данные:**
   - Используйте точные логины и пароли из раздела "Вход в систему"
   - Убедитесь, что выбрана правильная роль

2. **Проверьте консоль браузера:**
   - Откройте DevTools (F12)
   - Перейдите на вкладку Console
   - Посмотрите на детали ошибки

3. **Проверьте backend логи:**
   - В терминале, где запущен backend, должны быть логи запросов

## 🐛 Устранение неполадок

### Проблема: Не удается войти в систему

1. **Проверьте API сервер:**
   - Убедитесь, что backend запущен на `http://localhost:8080`
   - Проверьте, что API endpoints доступны

2. **Проверьте тестовые данные:**
   - Используйте точные логины и пароли из раздела "Вход в систему"
   - Убедитесь, что выбрана правильная роль

3. **Очистите localStorage:**
   ```javascript
   localStorage.clear();
   ```

### Проблема: Страница не загружается

1. **Проверьте зависимости:**
   ```bash
   npm install
   ```

2. **Проверьте порт:**
   - Приложение должно запускаться на порту 3001
   - Убедитесь, что порт не занят другим процессом
